Build a production-ready SaaS web application called "LifeOS AI".

# PRODUCT VISION

LifeOS AI is an AI-powered execution assistant.

This is NOT:

* A traditional todo app
* A habit tracker
* A subjective progress tracker
* A goal progress bar app

The system tracks measurable work units and allows users to update progress naturally through chat.

The AI should only understand language.

The database should always be the source of truth.

All calculations must happen in backend code.

---

# DATABASE DESIGN

Table: users

* id
* email
* created_at

Table: tasks

* id
* user_id
* title
* target_value
* completed_value
* unit
* due_date
* status
* created_at
* updated_at

Table: activity_logs

* id
* user_id
* task_id
* operation
* amount
* user_message
* extracted_json
* created_at

Activity logs are the source of truth.

Every update must create a new activity log record.

Never silently modify progress.

---

# EVENT SOURCING REQUIREMENT

IMPORTANT:

Do not directly overwrite progress without logging an event.

Every action creates an event.

Examples:

add 20
add 5
subtract 2
set 30
reset

Example:

Activity Log

add 20
add 5
subtract 2

Result:

23 completed

This enables:

* Audit history
* Undo support
* Analytics
* Transparency
